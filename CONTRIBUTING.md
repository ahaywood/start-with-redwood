First off, thank you for considering contributing to Start with Redwood! It's people like you that make this project possible and help it grow.

The following is a set of guidelines for contributing to Start with Redwood. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## Code of Conduct

This project and everyone participating in it is governed by the Start with Redwood Code of Conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to [amy@redwoodjs.com](mailto:amy@redwoodjs.com).

## How Can I Contribute?

### Reporting Bugs

This section guides you through submitting a bug report. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

Before submitting a bug report, please check the [issue tracker](https://github.com/ahaywood/start-with-redwood/issues) to see if the issue has already been reported.

#### How Do I Submit a Bug Report?

1. **Open a new issue**: Fill out the issue template with as much detail as possible.
2. **Provide steps to reproduce**: Include specific details such as the environment, versions, and steps to reproduce the issue.
3. **Provide a detailed description**: Explain what you expected to happen and what actually happened.

### Suggesting Enhancements

This section guides you through submitting an enhancement suggestion, including completely new features and minor improvements to existing functionality.

Before suggesting an enhancement, please check the [issue tracker](https://github.com/ahaywood/start-with-redwood/issues) to see if a similar idea has already been proposed.

#### How Do I Submit an Enhancement Suggestion?

1. **Open a new issue**: Use the enhancement issue template.
2. **Provide context**: Explain the current state and why this enhancement would be useful.
3. **Be clear and concise**: Describe the solution you’d like to see.

### Your First Code Contribution

Unsure where to begin contributing to Start with Redwood? You can start by looking through these "good first issue" and "help wanted" issues:

- [Good First Issues](https://github.com/ahaywood/start-with-redwood/issues?q=is:open+is:issue+label:%22good+first+issue%22) - issues which should only require a few lines of code, and a test or two.
- [Help Wanted Issues](https://github.com/ahaywood/start-with-redwood/issues?q=is:open+is:issue+label:%22help+wanted%22+) - issues that have a bit more complexity.

### Pull Requests

The process described here has several goals:

- Maintain the project’s quality.
- Fix problems that are important to users.
- Engage the community in working towards the best possible Start with Redwood.
- Enable a sustainable system for the project maintainers to review contributions.

Please follow these steps to have your contribution considered by the maintainers:

1. **Fork the repo and create your branch** from `main`.
2. **Make your changes** in a new branch:
    ```sh
    git checkout -b my-fix
    ```
3. **Test your changes** thoroughly.
4. **Create a pull request**:
    - Explain the issue that your PR addresses.
    - Ensure that your PR adheres to the project's coding standards.
    - Link to any related issues.

## Style Guides

### Commit Messages

- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.
- Reference issues and pull requests liberally after the first line.
- Use [Gitmoji](https://gitmoji.dev/).

### Code Style

- Ensure your code passes the linters.
- Include comments where necessary, especially for complex code.

### Documentation Style

- Use clear and concise language.
- Follow the same grammatical rules for comments and documentation as you would for any written content.
- Keep documentation up to date with any code changes.
